# 用户中心页面设计方案 v1.0

## 一、文档目标

本文档用于定义 TechCraft 项目的用户中心页面（客户版和开发者版）的页面结构、功能模块、数据组织和实现约束，作为 `profile-client.html` 和 `profile-developer.html` 的设计与前端实现依据。

---

## 二、需求理解

### 2.1 核心需求

用户中心是登录后的用户专属页面，提供以下核心功能：

1. **个人资料管理**：展示和编辑用户基本信息、头像、简介等
2. **任务管理**：
   - 客户：查看和管理自己发布的任务
   - 开发者：查看和管理自己参与/竞标的任务
3. **消息中心**：系统通知 + 用户私信 + 未读消息计数
4. **权限控制**：必须登录后才能访问，根据角色显示不同页面

### 2.2 用户角色区分

| 角色 | 页面文件 | 主要功能 |
|------|----------|----------|
| 客户 (client) | `profile-client.html` | 发布任务、管理我的任务、查看竞标 |
| 开发者 (developer) | `profile-developer.html` | 浏览任务、参与竞标、管理我的参与 |

### 2.3 与现有系统集成

- **认证系统**：复用现有的 `auth-state.js`、`auth-guard.js`、`auth-service.js`
- **样式系统**：复用 `base.css`、`components.css`，新增 `profile.css`
- **任务系统**：复用 `task-config.js`、`task-service.js`
- **布局系统**：复用 `shared-layout.js`

## 三、技术方案

### 3.1 技术栈

- **前端框架**：原生 JavaScript (ES6+ 模块)
- **样式**：CSS3 (CSS 变量、Flexbox、Grid)
- **数据存储**：LocalStorage/SessionStorage (认证信息) + Mock 数据 (开发阶段)
- **HTTP 请求**：Fetch API (预留真实 API 接口)

### 3.4 API 设计 (混合方式)

#### 3.4.1 开发阶段 (Mock)

```javascript
// user-service.js
const USE_MOCK = true; // 开关，方便切换

export async function getUserProfile(userId) {
  if (USE_MOCK) {
    return mockUserProfile(userId);
  }
  return fetchAPI(`/api/users/${userId}`);
}
```

#### 3.4.2 预留真实 API 接口

```
GET    /api/users/:id              # 获取用户资料
PUT    /api/users/:id              # 更新用户资料
GET    /api/users/:id/tasks        # 获取我的任务
GET    /api/users/:id/messages     # 获取消息列表
PUT    /api/messages/:id/read      # 标记消息已读
DELETE /api/messages/:id           # 删除消息
POST   /api/tasks/:id/apply        # 竞标任务
PUT    /api/tasks/:id/close        # 关闭任务
```

## 五、功能详细设计

### 5.1 个人资料模块

#### 5.1.1 资料卡片展示

**显示内容**：
- 头像 (默认头像或用户上传)
- 姓名
- 角色标签 (客户/开发者)
- 简介 (最多显示 3 行，超出截断)
- 实名认证状态徽章

**交互**：
- 点击「编辑资料」打开模态框
- 模态框包含：头像上传、姓名、简介、地区、网站等字段

#### 5.1.2 资料编辑模态框

**表单字段**：
- 头像上传 (支持拖拽、点击上传)
- 姓名 (必填，2-20字符)
- 简介 (选填，最多200字)
- 地区 (选填)
- 个人网站 (选填，URL格式校验)
- 技能标签 (开发者专用，最多选择5个)
- 公司名称 (客户专用)
- 职位 (客户专用)

**校验规则**：
- 姓名长度：2-20字符
- 简介长度：0-200字符
- 网站格式：有效的 URL

### 5.2 任务管理模块

#### 5.2.1 客户版 - 我发布的任务

**Tab 切换**：
- 全部任务
- 招募中
- 进行中
- 已完成
- 已关闭

**任务操作**：
- 编辑任务 (仅招募中状态)
- 关闭任务 (招募中/进行中状态)
- 查看竞标 (招募中状态，显示竞标人数和未读数)
- 查看详情 (所有状态)
- 重新发布 (已关闭状态)

**任务卡片字段**：
- 任务标题
- 任务简介
- 预算范围
- 截止时间
- 状态标签
- 竞标数 / 未读竞标数
- 雇佣的开发者 (进行中/已完成)

#### 5.2.2 开发者版 - 我参与的任务

**Tab 切换**：
- 全部参与
- 竞标中
- 已接受
- 已完成
- 已拒绝

**任务操作**：
- 查看详情 (所有状态)
- 撤销竞标 (竞标中状态)
- 标记完成 (已接受状态，需要客户确认)
- 收藏/取消收藏 (所有状态)

**任务卡片字段**：
- 任务标题
- 任务简介
- 预算范围
- 截止时间
- 状态标签
- 我的竞标状态
- 我的报价
- 竞标时间

### 5.3 消息中心模块

#### 5.3.1 消息类型

| 类型 | 说明 | 图标 |
|------|------|------|
| system | 系统通知 (任务状态变更、实名认证结果等) | 铃钟 |
| task | 任务相关 (新竞标、被雇佣等) | 文档 |
| private | 用户私信 | 信封 |

#### 5.3.2 消息列表

**筛选**：
- 全部消息
- 未读消息
- 系统通知
- 任务消息
- 私信

**列表项显示**：
- 消息类型图标
- 消息标题 (未读加粗)
- 消息内容预览 (最多显示一行)
- 时间 (相对时间：刚刚/5分钟前/今天/昨天)
- 未读红点

**操作**：
- 点击消息打开详情
- 标记已读/未读
- 删除消息

#### 5.3.3 未读消息计数

- 在导航栏用户头像上显示红点 + 数字
- 跨标签页同步
- 点击消息后自动标记已读并更新计数

### 5.4 概览仪表盘

#### 5.4.1 客户版概览

**统计卡片**：
- 发布任务数 (招募中 X / 进行中 X / 已完成 X)
- 总预算支出
- 平均任务完成时间
- 响应率

**最近任务**：
- 显示最近 3 条任务
- 快捷操作按钮

**最新消息**：
- 显示最近 3 条未读消息
- 点击跳转到消息中心

#### 5.4.2 开发者版概览

**统计卡片**：
- 参与任务数 (竞标中 X / 进行中 X / 已完成 X)
- 总收入估算
- 平均评分
- 成交率

**最近参与**：
- 显示最近 3 条参与的任务
- 快捷操作按钮

**最新消息**：
- 显示最近 3 条未读消息
- 点击跳转到消息中心

## 六、风险点

### 6.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Mock 数据与真实 API 数据结构不一致 | 后续集成时需要大量修改 | 严格按照真实 API 预期的数据结构设计 Mock |
| 跨标签页状态同步失败 | 用户在多个标签页操作时数据不一致 | 使用 storage 事件 + 心跳检测双重保障 |
| 大量消息数据导致页面卡顿 | 消息列表渲染缓慢 | 实现分页加载 + 虚拟滚动 |
| 头像上传文件大小限制 | 用户体验差 | 前端压缩 + 大小限制 + 格式校验 |

### 6.2 用户体验风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 页面加载时间长 | 用户流失 | 骨架屏 + 懒加载 |
| 操作反馈不及时 | 用户重复操作 | 加载状态 + 禁用按钮 |
| 移动端侧边栏遮挡内容 | 操作困难 | 抽屉式侧边栏 + 遮罩点击关闭 |
| 未读消息计数不准确 | 用户错过重要消息 | 定时轮询 + 跨标签页同步 |

### 6.3 安全风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 用户资料被越权修改 | 数据泄露 | 后端验证权限 + 前端路由守卫 |
| XSS 攻击 (简介、网站字段) | 用户数据被盗 | 所有用户输入通过 textContent 注入 |
| CSRF 攻击 | 未授权操作 | Token 认证 + 请求头验证 |

## 七、边界情况

### 7.1 认证相关

| 场景 | 处理方式 |
|------|----------|
| 未登录访问 | 自动跳转到登录页，携带目标 URL |
| 登录后访问错误角色页面 | 跳转到正确的角色页面 |
| Token 过期 | 自动刷新 Token，失败则退出登录 |
| 实名认证状态不满足要求 | 显示提示，禁止特定操作 |

### 7.2 数据相关

| 场景 | 处理方式 |
|------|----------|
| 用户资料为空 | 显示默认值 + 引导编辑 |
| 任务列表为空 | 显示空状态插图 + 引导去任务大厅 |
| 消息列表为空 | 显示空状态插图 + 提示暂无消息 |
| 头像加载失败 | 显示默认头像 |

### 7.3 操作相关

| 场景 | 处理方式 |
|------|----------|
| 网络请求失败 | 显示错误提示 + 重试按钮 |
| 编辑资料时取消 | 不保存修改，关闭模态框 |
| 删除消息时误操作 | 显示二次确认弹窗 |
| 同时在多个标签页操作 | 跨标签页同步状态 |

### 7.4 性能相关

| 场景 | 处理方式 |
|------|----------|
| 消息列表超过 100 条 | 分页加载，每页 20 条 |
| 任务列表超过 50 条 | 分页加载，每页 10 条 |
| 头像文件过大 | 前端压缩后上传，限制 2MB |
| 页面加载慢 | 显示骨架屏，超时显示重试 |
