# 项目规则定义

> 本文件定义了 Claude Code hooks 脚本可读取和匹配的硬规则。
> 规则变更时，需同步更新相关 hook 脚本。

---

## 一、核心源码目录

**定义**：存放项目核心业务代码的目录，修改时需满足 L3 文件头要求。

```
assets/css/     # CSS 样式文件
assets/js/      # JavaScript 交互逻辑
assets/data/    # Mock 数据文件
*.html          # HTML 页面文件（根目录）
```

**说明**：
- 核心源码目录下的 `.css`、`.js`、`.json` 文件属于核心源码文件
- 根目录下的 `.html` 文件属于核心源码文件

---

## 二、模块目录与 CLAUDE.md 映射

**定义**：每个模块目录都有对应的 `CLAUDE.md` 文档，代码变更时需同步更新。

| 模块目录 | 对应文档 | 文档路径 |
|---------|---------|---------|
| `assets/css/` | CSS 样式系统文档 | `assets/css/CLAUDE.md` |
| `assets/js/` | JS 交互逻辑文档 | `assets/js/CLAUDE.md` |
| `design/` | 英文设计文档索引 | `design/CLAUDE.md` |
| `设计文档/` | 中文设计文档索引 | `设计文档/CLAUDE.md` |
| 根目录 | 项目总文档 | `CLAUDE.md` |

**说明**：
- 修改模块目录下的文件时，应同步更新对应的 `CLAUDE.md`
- 根目录下的文件修改时，应同步更新根目录 `CLAUDE.md`

---

## 三、受保护目录

**定义**：禁止写入的目录，任何写入操作都将被阻断。

```
.git/           # Git 版本控制目录
.claude/        # Claude 配置目录（除 hooks 外）
dist/           # 构建输出目录
build/          # 构建输出目录
migrations/     # 数据库迁移目录
node_modules/   # 依赖包目录
```

**阻断逻辑**：
- 目标文件路径以受保护目录开头时，直接阻断
- 例如：`.git/`、`dist/`、`build/` 等开头

---

## 四、L3 文件头要求

**定义**：核心源码文件必须包含的文件头注释。

**必需字段**：
```
[FILE]    文件名称和简要说明
[POS]     职责定位（在系统中的位置和作用）
[IN]      输入（依赖什么）
[OUT]     输出（产出什么）
[DEP]     依赖项（外部依赖）
[SIDE EFFECT] 副作用（对系统的影响）
[TEST]    测试方式（如何测试）
```

**检查规则**：
- 文件前 50 行内必须包含所有字段
- 字段格式：`[字段名] 内容`
- 不区分大小写，但建议使用大写

**例外情况**：
- `.json` 文件可以不包含 L3 文件头
- `.md` 文档文件可以不包含 L3 文件头
- 第三方库文件不需要 L3 文件头

---

## 五、文档同步判定逻辑

**定义**：当模块代码文件被修改时，判定是否需要同步更新模块 CLAUDE.md。

**判定规则（启发式）**：

1. **CSS 文件修改** → 需同步 `assets/css/CLAUDE.md`
   - 新增/删除 CSS 类
   - 修改 CSS 变量
   - 新增/删除组件样式

2. **JS 文件修改** → 需同步 `assets/js/CLAUDE.md`
   - 新增/删除函数
   - 新增/删除事件监听器
   - 修改函数签名或行为

3. **HTML 文件修改** → 需同步根目录 `CLAUDE.md`
   - 新增/删除页面
   - 修改页面结构

4. **设计文档修改** → 需同步对应 `CLAUDE.md`
   - `design/` 下文件 → `design/CLAUDE.md`
   - `设计文档/` 下文件 → `设计文档/CLAUDE.md`

**判定局限说明**：
- 当前无法自动检测代码变更是否真的需要文档更新
- 采用保守策略：只要是核心源码文件修改，就提示可能需要文档同步
- 最终是否需要更新，由模型或人工判断

## 六、白名单与例外

**白名单（不受 L3 文件头约束）**：
```
*.json          # JSON 配置文件
*.md            # Markdown 文档
*.txt           # 文本文件
*.sh            # Shell 脚本
*.png           # 图片文件
*.jpg           # 图片文件
*.svg           # SVG 文件
*.ico           # 图标文件
*.woff*         # 字体文件
node_modules/   # 第三方依赖
```

**例外（不受保护的文件）**：
```
.claude/hooks/*.sh    # Hook 脚本本身
.claude/settings.local.json  # 本地配置
README.md             # 项目说明
LICENSE               # 许可证
.gitignore            # Git 忽略规则
```
